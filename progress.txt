What needs to be built — the full AI stack

There are three distinct components:
Component 1 — Recommendation Engine (on-device, SQLite-driven)

A collaborative filtering algorithm running locally in Dart. No Gemini needed for this — it uses the data already in SQLite:

    Skill-match scoring — cosine similarity between a user's skills array and each post's required skills
    Collaborator ranking — weighted graph: shared skills + shared faculty + mutual follows
    Opportunity matching — skill overlap score against opportunity requirements + year-of-study filter

This runs entirely offline. The results are cached in a recommendations SQLite table.
Component 2 — Gemini API integration (GeminiService)

Calls gemini-1.5-flash via Dio for:

    Generating the personalized nudge messages (streak, collaboration, badge progress)
    Summarising a user's profile into a bio suggestion on edit profile
    Generating the AI reasoning text shown in the suspicion score detail screen

Component 3 — AI Suspicion Scoring (admin-side)

A rule-based + Gemini hybrid that runs when content is flagged:

    Rule-based pre-filter checks: external links, known watermark patterns, file types
    Gemini classifies the text/description for the four categories (inappropriate, hate speech, spam, external link risk)
    Score is stored in posts.suspicion_score column (already in the schema)
